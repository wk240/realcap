# Lending Platform Fraud Prevention Case Study

A leading lending platform implemented RealCap to verify borrower financial screenshots, resulting in an 85% reduction in fraud losses. This article details the challenges faced, solution implementation, and quantifiable results achieved.

## Industry Background

The lending industry has experienced rapid growth, but accompanying fraud risks have become increasingly severe. According to industry reports, **annual losses from screenshot fraud exceed $2 billion**. Borrowers obtain loans they cannot repay by forging bank balance screenshots and income proof screenshots, causing direct financial losses to platforms and affecting the healthy development of the entire industry.

Traditional manual verification methods have clear defects:

- **Low efficiency**: Manual review of each document takes 5-10 minutes, unable to handle high-frequency applications
- **Insufficient accuracy**: Manual identification of forged screenshots is only 60-70% accurate
- **High costs**: Requires a large number of reviewers, labor costs continue to rise
- **Standardization difficulties**: Different reviewers have inconsistent standards, prone to missed judgments and false judgments

## Challenges Faced by the Platform

This leading lending platform processed over 500,000 loan applications per month, with approximately 15% involving financial screenshot verification. The platform faced the following specific problems:

### 1. High Default Rate Problem

The platform found that **40% of defaulting users submitted suspicious financial screenshots** during application. These users obtained loan amounts far beyond their actual repayment capacity by forging bank balances or income proof.

### 2. AI Forgery Technology Upgrade

With the development of AI technology, the quality of forged screenshots has significantly improved. Traditional Photoshop editing traces can be detected through image analysis, but AI-generated fake screenshots are almost impossible to distinguish visually. The platform review team reported:

> "Now forged screenshots are becoming more realistic, many with fonts and colors exactly matching real screenshots. Manual review is completely unable to distinguish them."

### 3. Approval Efficiency Bottleneck

The traditional verification process required manual review of each financial document, averaging over 3 working days. This not only affected user experience but also limited the platform's business expansion capability.

### 4. Rising Risk Control Costs

To address fraud risk, the platform continuously added review staff, with annual risk control labor costs exceeding 20 million yuan, but results remained unsatisfactory.

## Solution Implementation

The platform began evaluating screenshot verification technology solutions in Q1 2025, ultimately choosing RealCap as a partner. The implementation process consisted of three phases:

### Phase 1: Technical Evaluation and Pilot (1 month)

- Evaluated RealCap's technical principles and verification process
- Tested verification effectiveness in small-scale application batches
- Conducted API integration testing with existing risk control systems

**Pilot Results**: In 1,000 test applications, RealCap successfully identified 47 forged screenshots with 99.2% accuracy.

### Phase 2: System Integration (2 months)

- Developed API interfaces to connect with RealCap verification services
- Embedded screenshot verification in the application process
- Designed exception handling and manual review processes
- Completed review team training

**Technical Architecture**:

```
Application Process:
User Submission → RealCap Screenshot Upload → Automatic Verification → Result Return → Approval Decision
              ↑                                        ↓
```

### Phase 3: Full Deployment (1 month)

- Enabled screenshot verification in all loan products
- Established real-time monitoring and warning mechanisms
- Continuously optimized verification parameters and thresholds

### Risk Identification Details

Main fraud types identified by the RealCap system during actual operation:

1. **Bank Balance Forgery (55%)**
   - AI-generated fake bank app interfaces
   - Modified real screenshot balance numbers
   - Using others' real screenshots impersonation

2. **Income Proof Forgery (30%)**
   - Forged payroll screenshots
   - Modified real income screenshot amounts
   - Fabricated non-existent employers

3. **Asset Proof Forgery (15%)**
   - Forged investment account screenshots
   - Modified real estate certificates and other asset proofs
   - AI-generated fake proof documents

## Technical Principles

RealCap verification system adopts a multi-layer verification mechanism:

### 1. Screenshot Source Verification

Record key metadata when capturing screenshots on user device end:

- Device unique identifier
- Screenshot timestamp
- System environment information
- App context data

This metadata is encrypted and signed, cannot be added or modified afterward, ensuring screenshot source authenticity.

### 2. Content Integrity Verification

Analyze logical consistency of screenshot content:

- Whether number format matches bank app standards
- Whether timestamp is reasonable
- Whether data logic is coherent
- Whether image details are natural

### 3. AI Forgery Detection

Use deep learning models to detect AI generation features:

- Image texture analysis
- Font consistency detection
- Lighting and shadow rationality verification
- Metadata integrity check

## Best Practice Recommendations

Based on this implementation experience, we summarize the following best practices for other lending platforms:

### 1. Progressive Deployment

Recommend starting from small-scale pilots, gradually expanding verification scope. This can:

- Reduce deployment risk
- Accumulate verification data to optimize parameters
- Allow team to gradually adapt to new processes

### 2. Retain Manual Review Mechanism

Although automated verification accuracy is high, still recommend retaining manual review mechanism for edge cases. Platform settings:

- Automatic verification pass rate threshold: 95%
- Abnormal applications enter manual review
- Regular audit of verification results

### 3. Deep Integration with Risk Control System

Screenshot verification should be used synergistically with other risk control means:

- Combined with user credit data
- Added behavioral analysis models
- Integrated into comprehensive risk assessment

### 4. Continuous Monitoring Optimization

Establish verification effectiveness monitoring mechanism:

- Track association between defaulting users and verification results
- Analyze missed and false judgment cases
- Regularly optimize verification parameters

## Summary

This lending platform's case proves that screenshot verification technology can effectively solve financial document fraud problems faced by the lending industry. Through RealCap verification system, the platform not only significantly reduced fraud losses, but also notably improved operational efficiency and user experience.

For lending platforms facing similar problems, recommend evaluating and deploying screenshot verification technology early. In the context of increasingly mature AI forgery technology, traditional verification methods are difficult to cope with new risk challenges.

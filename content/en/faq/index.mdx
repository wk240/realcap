## Screenshot Verification Basics

### What is screenshot verification?

Screenshot verification is a technology that confirms screenshot authenticity, verifying whether screenshots are genuinely captured, unaltered, and not AI-generated fake images.

RealCap adopts a "trusted source" verification philosophy: capturing metadata (device information, timestamp, operation context) at the moment of screenshot capture and encrypting with signature. This metadata cannot be retroactively added or modified, fundamentally eliminating AI forgery possibilities.

**Difference from Traditional Verification:**

| Traditional Method | RealCap Method |
|-------------------|----------------|
| Post-analysis of screenshot edits | Trusted record created at capture |
| Cannot detect AI "natively" generated | AI cannot forge signed metadata |
| 60-70% accuracy | Over 99% accuracy |
| Relies on manual judgment | Automated rapid verification |

### How does RealCap detect AI-generated fake screenshots?

RealCap employs multi-layer detection mechanisms:

**Layer 1: Source Trust Verification**

Screenshots captured through RealCap automatically generate encrypted signatures. AI-generated screenshots cannot forge this signature and are immediately identified as untrusted source during verification.

**Layer 2: Metadata Analysis**

Real screenshots contain complete system metadata (device info, timestamp, screen parameters, etc.). AI-generated screenshots often lack or show inconsistent metadata.

**Layer 3: AI Feature Detection**

Our deep learning model specifically detects AI-generated image features:

- Image texture pattern anomalies
- Unnatural font rendering
- Unreasonable lighting and shadows
- UI element layout deviations

**Layer 4: Data Logic Verification**

Analyzes whether screenshot data is logically consistent:

- Number format standardization
- Timestamp rationality
- Data variation continuity

### Can RealCap verify screenshots taken before installation?

**No.**

RealCap's verification mechanism relies on metadata signature at screenshot capture moment. Only screenshots captured through RealCap (app, browser extension, or SDK) can obtain valid verification signatures.

Screenshots taken before RealCap installation cannot retroactively add signatures, thus cannot pass source trust verification.

**Alternative Solution:**

For historical screenshots that cannot be recaptured, RealCap provides **Forgery Detection Analysis** service (without source verification), which can detect obvious edit tampering and some AI generation features, but with relatively lower accuracy (approximately 85%).

## Integration Questions

### How do I integrate RealCap API?

RealCap provides RESTful API for quick integration into any system.

**Integration Steps:**

**Step 1: Obtain API Key**

Create project in RealCap console, obtain API Key and Secret Key.

**Step 2: Configure Authentication**

```javascript
const headers = {
  'Authorization': 'Bearer YOUR_API_KEY',
  'Content-Type': 'application/json'
};
```

**Step 3: Send Verification Request**

```javascript
const response = await fetch('https://api.realcap.app/v1/verify', {
  method: 'POST',
  headers: headers,
  body: JSON.stringify({
    verificationType: 'bank_balance',
    screenshot: base64EncodedImage,
    metadata: { declaredAmount: 100000 }
  })
});
```

**Step 4: Process Verification Result**

```javascript
const result = await response.json();
if (result.status === 'verified' && result.fraudProbability < 0.1) {
  // Verification passed, continue business process
} else {
  // Verification failed or needs review, handle exception
}
```

### Is there a webhook callback option?

**Yes.**

RealCap supports webhook callback notification after verification completion, suitable for asynchronous verification result processing.

**Configure Webhook:**

Configure callback URL in console settings:

```
https://your-platform.com/webhook/realcap
```

**Callback Request Format:**

```json
{
  "verificationId": "verify_abc123",
  "status": "verified",
  "fraudProbability": 0.02,
  "confidence": 0.99,
  "timestamp": "2026-04-30T10:00:00Z",
  "signature": "sha256_hmac_signature"
}
```

**Verify Signature:**

```javascript
const verifySignature = (payload, signature, secretKey) => {
  const expectedSignature = crypto
    .createHmac('sha256', secretKey)
    .update(JSON.stringify(payload))
    .digest('hex');
  return signature === expectedSignature;
};
```

### Is batch verification supported?

**Yes.**

RealCap provides dedicated batch verification API, supporting multiple screenshots submitted at once for verification.

**Batch Verification Request:**

```javascript
const response = await fetch('https://api.realcap.app/v1/verify/batch', {
  method: 'POST',
  headers: headers,
  body: JSON.stringify({
    verifications: [
      { verificationType: 'bank_balance', screenshot: screenshot1 },
      { verificationType: 'income_proof', screenshot: screenshot2 },
      { verificationType: 'identity', screenshot: screenshot3 }
    ],
    callbackUrl: 'https://your-platform.com/webhook'
  })
});

// Returns batch task ID
const batchResult = await response.json();
// { batchId: 'batch_xyz123', status: 'processing', totalCount: 3 }
```

**Batch Verification Features:**

- Mixed type verification support
- Asynchronous processing, non-blocking main flow
- Webhook callback completion notification

## Privacy & Security

### How is user screenshot data handled?

RealCap strictly adheres to data privacy principles:

**Data Processing Principles:**

| Principle | Practice |
|-----------|----------|
| Minimal collection | Only collect verification-essential metadata |
| No original storage | Screenshots only for analysis, deleted after verification |
| Result minimization | Only return probability scores, no sensitive content |
| User control | Users can delete verification records anytime |

**Data Lifecycle:**

```
Screenshot upload -> Analysis processing -> Result generation -> Original deletion -> Record retention (7 days)
      ↓                   ↓                   ↓                   ↓                   ↓
 Encrypted transmission  In-memory processing  Probability output  Immediate deletion  User deletable
```

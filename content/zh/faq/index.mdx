## 截图验证基础

### 什么是截图验证？

截图验证是一种确认截图真实性的技术，验证截图是否是真实拍摄的、未被篡改、也不是由AI生成的虚假图像。

RealCap采用"源头可信"的验证理念：在截图拍摄时即时捕获元数据（设备信息、时间戳、操作上下文），并加密签名。这些元数据无法事后添加或修改，从根本上杜绝AI伪造截图的可能性。

**与传统验证的区别：**

| 传统方式 | RealCap方式 |
|---------|------------|
| 事后分析截图是否被编辑 | 拍摄时即建立可信记录 |
| 无法检测AI"原生"生成 | AI无法伪造签名元数据 |
| 准确率60-70% | 准确率99%以上 |
| 依赖人工判断 | 自动化快速验证 |

### RealCap如何检测AI生成的虚假截图？

RealCap采用多层检测机制：

**第一层：来源可信验证**

通过RealCap拍摄的截图会自动生成加密签名。AI生成的截图无法伪造这个签名，验证时会立即被识别为不可信来源。

**第二层：元数据分析**

真实截图包含完整的系统元数据（设备信息、时间戳、屏幕参数等）。AI生成的截图往往缺失或不一致。

**第三层：AI特征检测**

我们的深度学习模型专门检测AI生成图像的特征：

- 图像纹理模式异常
- 字体渲染不自然
- 光照和阴影不合理
- 界面元素布局偏差

**第四层：数据逻辑验证**

分析截图中的数据是否符合逻辑：

- 数字格式是否标准
- 时间戳是否合理
- 数据变动是否连贯

### RealCap可以验证安装前拍摄的截图吗？

**不可以。**

RealCap的验证机制依赖于截图拍摄时的元数据签名。只有通过RealCap（应用、浏览器扩展或SDK）拍摄的截图才能获得有效的验证签名。

安装RealCap之前拍摄的截图无法事后添加签名，因此无法通过来源可信验证。

**替代方案：**

对于无法重新拍摄的历史截图，RealCap提供**伪造检测分析**服务（不含来源验证），可检测明显的编辑篡改和部分AI生成特征，但准确率相对较低（约85%）。

## 集成问题

### 如何集成RealCap API？

RealCap提供RESTful API，可快速集成到任何系统。

**集成步骤：**

**Step 1：获取API密钥**

在RealCap控制台创建项目，获取API Key和Secret Key。

**Step 2：配置认证**

```javascript
const headers = {
  'Authorization': 'Bearer YOUR_API_KEY',
  'Content-Type': 'application/json'
};
```

**Step 3：发送验证请求**

```javascript
const response = await fetch('https://api.realcap.app/v1/verify', {
  method: 'POST',
  headers: headers,
  body: JSON.stringify({
    verificationType: 'bank_balance',
    screenshot: base64EncodedImage,
    metadata: { declaredAmount: 100000 }
  })
});
```

**Step 4：处理验证结果**

```javascript
const result = await response.json();
if (result.status === 'verified' && result.fraudProbability < 0.1) {
  // 验证通过，继续业务流程
} else {
  // 验证失败或需复核，处理异常
}
```

### 有Webhook回调选项吗？

**是的。**

RealCap支持验证完成后的Webhook回调通知，适合异步处理验证结果。

**配置Webhook：**

在控制台设置中配置回调URL：

```
https://your-platform.com/webhook/realcap
```

**回调请求格式：**

```json
{
  "verificationId": "verify_abc123",
  "status": "verified",
  "fraudProbability": 0.02,
  "confidence": 0.99,
  "timestamp": "2026-04-30T10:00:00Z",
  "signature": "sha256_hmac_signature"
}
```

**验证签名：**

```javascript
const verifySignature = (payload, signature, secretKey) => {
  const expectedSignature = crypto
    .createHmac('sha256', secretKey)
    .update(JSON.stringify(payload))
    .digest('hex');
  return signature === expectedSignature;
};
```
